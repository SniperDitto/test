Markup의 등장 배경
문서 공유 시 인터넷 속도가 느려 공유가 불편함
-> 일종의 형식에 따라 데이터를 전송해 받는 쪽에서 형식에 따라 해석
-> 데이터 자체만 전송
-> 모든 브라우저에서 공통적으로 해석할 수 있는 형식 필요
-> 데이터의 구분 수단

미들웨어를 활용해 데이터 전달

웹 브라우저의 주소창에 입력해서 접속
-> 요청을 보내다(request)

html문서를 더블클릭해 열기(웹브라우저로)
-> 웹 서버에 text.html을 배포하다

구글의 Google Maps가 웹 분야에 큰 변화를 줌
-> Web 2.0 시대
-> 자바스크립트
-> 프론트엔드

Markup의 특징
1. 태그를 열면 닫아야 한다. (tag) -> single 태그 제외 <br/>
2. 원소들로 이루어져 있다. (element/tag/object)



Object : 객체

객체 = 속성 + 움직임
