
웹 서버에 html 파일을 저장
-> 웹 서버에서 내 컴퓨터로
-> 내 브라우저가 내용을 해석해 보여줌

서버 : 특정 용도로 컴퓨터를 여러 명이 동시에 사용하는 환경
	외부에서 최초 접속 시 root 경로의 위치로 접속
	
상대경로 : 내 위치를 기준으로 원하는 위치 표시, '/' 사용
		( ex) ./img/1.jpg)

이미지
-> img(single tag)
	웹에서 사용할 수 있는 이미지 확장자 : png, jpg, gif
	우클릭 시 다른 이름으로 저장 가능
	- src
		폴더 디렉터리 경로 이동 
			cd .. : 상위 폴더로(change directory)
			절대 경로 입력 : ex) C:\Users\PKNU\Desktop\2022JAVA_YSH\20220106
			웹 서버 개발 시 상대경로를 이용
	- width
	- height
	- alt (마우스 오버 시 뜨는 글자)
-> 

이미지 레이아웃 구성 시 table 사용하지 않는 이유
-> 변경 사항 발생 시 수정하기가 힘들다
-> 큰 틀 역할을 할 표를 만들고 그 안에 작은 표를 만들어 구성 (메뉴 표 따로, 내용 표 따로 ...)
-> 표는 객체처럼 수정이 가능하지만 표의 셀은 인식이 불가하므로
-> 주석으로 메뉴와 내용 등 시작과 끝을 알기 쉽게 표시 (보안상 주석 쓰지 않는 경우도 있다)
-> br 태그 사용 X
-> text만 달랑 사용하지 말 것 (span, div 등 어딘가에 담아서 표시)
-> 공백 부분도 테이블 내용에 포함되게 하는 것이 좋다

가로 수평방향 정렬 horizontal
세로 수직방향 정렬 vertical


예약어
&gt; : > grater than
&lt; : < less than
&nbsp; : " "(공백 한칸)




웹에서의 페이지 이동(링크)
a태그를 이용 + 상대경로(자신의 위치 기준으로)
